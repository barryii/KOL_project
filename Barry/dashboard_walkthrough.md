# 儀表板堆疊直條圖升級導覽 (Stacked Bar Update)

我們已經根據您的需求，將首頁的比較看板進行了大幅度的視覺與資料結構升級。

## 視覺與互動升級重點

1. **滿版網格 (Full Width Layout)**
   - 移除原先被擠在左右兩側的設計。現在「每月總觀看數走勢」等大圖表各自佔據全寬版面，擁有更大的空間來顯示堆疊圖。

2. **直條圖三種比例同時呈現 (Stacked Bar Charts)**
   - 我們保留了原先配色的基底（YTer A 為亮藍色、YTer B 為紫色）。
   - 將 **一般影片**、**Shorts**、**直播** 的總計數字全部拆出來，以長條圖「疊加」的方式呈現。
   - > [!TIP]
     > 最上方的「四大天王數據卡」，包含「累積觀看次數」以及「總互動數」直條圖也全面改成三種顏色疊加的設計了，讓您不但能看出總量的差距，還能一眼看出誰是靠 Shorts 取勝。

3. **「光束」輔助線 (Hover Light Beam)**
   - 您提到的需求：滑鼠移動時應該有一條垂直的線段幫助對準。
   - 我們為 Chart.js 寫了一套微型的 `hoverLinePlugin`，現在只要滑鼠在任何 X 軸區間遊走，就會透出一道 **半透明的垂直白色輔助光**，並且列出包含 Shorts、直播等所有組成成分的數值。

## 資料結構重構 (後端)
- 完全改寫了 `front_app.py` 的 `/api/overview`，我們現在會在 SQL Query 中帶入 `GROUP BY type`，不僅大幅增加了 API 提供數據的解析度，也將整體速度維持在最佳狀態。
- 為了讓「平均化」的數學問題合理，系統在後端特別計算了一組 `all` 模組，專門供應給「平均觀看」這類不適合做堆疊加總的圖表。
